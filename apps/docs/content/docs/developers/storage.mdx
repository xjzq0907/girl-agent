Никакой БД. Только файлы. Контракт — `src/storage/md.ts`.

## DATA_ROOT

```ts
export const DATA_ROOT = resolveDataRoot();

function resolveDataRoot(): string {
  if (process.env.GIRL_AGENT_DATA) return process.env.GIRL_AGENT_DATA;
  if (process.platform === "win32") return path.join(process.env.APPDATA ?? os.homedir(), "girl-agent", "data");
  return path.join(os.homedir(), ".local", "share", "girl-agent", "data");
}
```

Один путь на установку. Все профили — поддиректории.

## relationship.md — формат

```md
---
score:
  interest: 50
  trust: 30
  attraction: 40
  annoyance: 15
  cringe: 10
stage: convinced
conflict:
  level: 0
  apologized: false
meters:
  rudeness: 12
  flakiness: 25
  inconsistency: 0
  pressure: 8
  jealousy: 5
lastUpdated: 2026-05-16T12:00:00Z
---

# отношения с {name}

(свободный текст, который ведёт LLM — что между ними сейчас)
```

Score колапс из 5 шкал — `updateScore(slug, delta)` — нормализует delta в `[-N..+N]` и кламп.

## Атомарность

- **Тонкие записи** идут через `appendFile` (логи) — это норма.
- **Толстые конфигурации** (`config.json`, `relationship.md`) пишутся **в темп файл** и переименовываются (atomic rename на одной файловой системе).
- Параллельная запись разных профилей — безопасна (директории не пересекаются).
- Параллельная запись одного и того же файла одного профиля — **не защищена.** Поэтому Runtime сериализует записи во времени (нет «двух хендлеров на один профиль»).

## Конвенции

- **Имена в slug'ах.** Папка профиля = `slugify(name)`. UI ругается, если slug уже занят.
- **Никогда не храните секреты в `persona.md` / `relationship.md`.** Они идут в LLM-промпт. Секреты — в `config.json` и **никогда** в memory/long-term.
- **Дата сессий — в её tz.** Файл `log/2026-05-16.md` — это её 16 мая, а не UTC.
- **`memory/episodes/*` пишет LLM**, не пользователь. Имя файла — slugified ключ события.
