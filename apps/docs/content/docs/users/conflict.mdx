`girl-agent` не «безусловно добрая» — у неё есть **конфликт-движок**. Если ты её обидишь,
проигнорируешь несколько часов, нагрубишь — это попадёт в `conflict/unresolved.md` и она
будет вести себя соответствующе, пока вы это не закроете.

## Состояние отношений: 5 счётчиков

| Счётчик     | Что значит                                                  |
| ----------- | ----------------------------------------------------------- |
| interest    | насколько ей с тобой интересно (-100..100)                  |
| trust       | доверие, открытость (-100..100)                             |
| attraction  | влечение (-100..100)                                        |
| annoyance   | накопленное раздражение от твоих действий (-100..100)       |
| cringe      | сколько раз ты её **«закринжанул»** (-100..100)             |

Можно посмотреть:
- WebUI → профиль → **Overview**
- В чате: `:status`, `:relationship`, `:cringe`
- В файле: `relationship.md` (в HTML-комменте `<!--score:{...}-->`)

## Что повышает annoyance

- Долгий игнор с твоей стороны (не отвечал 4+ часа без причины)
- Перебивания, разговоры «по верхам»
- Прямая грубость
- Игнор её вопросов (она спрашивает, ты отвечаешь не на это)
- Push с твоей стороны (требуешь фото / личное / встречу до доверия)

## Что повышает cringe

- Очень кринжовые попытки флирта на ранних стадиях
- «Pickup-фразы» из 2010-х
- Слишком прямой sexting когда не готова
- Уменьшительные («малая», «крошка», «зайка») на стадии 1-3 от незнакомца
- Чрезмерные эмодзи

<Callout type="info">
  `cringeTolerance` зависит от стадии. На `dating-stable` (стадия 7) она многое терпит. На
  `tg-given-cold` (стадия 2) — почти всё кринж.
</Callout>

## Что повышает interest / trust / attraction

- Интересные диалоги, длинные истории с её стороны
- Хорошее чувство юмора
- Помнить, что она говорила раньше
- Реакция на её эмоции
- Уместные комплименты (не банальные)

Каждый ответ движок **оценивает через LLM** и применяет delta к счётчикам. Это всё видно в
`log/` файле в строках вида `mood:+5 interest, -1 annoyance`.

## Конфликты

Когда `annoyance` или `cringe` пересекают порог — открывается **конфликт**.

### Что происходит при конфликте

1. В `conflict/unresolved.md` появляется запись: «обиделась за то-то, ожидает извинений»
2. Поведение меняется:
   - Игнор-шанс растёт
   - Сообщения становятся короче и холоднее
   - Может «выйти» из диалога на сутки
   - В bot-режиме — просто молчит
   - В userbot-режиме — «не онлайн», прочитала и не ответила
3. В `log/` появляется системная отметка `[conflict] opened: ...`

### Как видеть текущие конфликты

- WebUI → **Conflict** (если есть аддон conflict-panel) или прямо открыть `conflict/unresolved.md`
- Команды: `:debug conflict` показывает текущие открытые

### Как закрывать

Конфликт **не закроется молча со временем**. Нужно:

1. **Признать** — извиниться, показать что понял в чём проблема
2. Сделать **компенсирующее действие** — комплимент, забота, изменение поведения
3. Если конфликт серьёзный (cringe > 60) — может потребоваться время + действия

LLM сам оценит сообщение «достаточно ли это для закрытия». Если да — запись переезжает из
`unresolved.md` в `conflict/log.md` с пометкой `closed`.

### Принудительное закрытие
Если знаешь что движок завис в неоправданном конфликте:

```
:debug close-conflict <id>
```

Или удалить запись руками из `conflict/unresolved.md`.

## Когда она бросает

В крайних случаях:
- `interest < -40` + `annoyance > 70` + долгий конфликт без закрытия
- Несколько повторений одного и того же триггера
- Прямая грубость / угрозы

→ стадия может **сдвинуться вниз** или даже до `dumped`. В `dumped` она:
- отвечает крайне редко (95% игнор)
- если отвечает — холодно, односложно
- не пишет первой
- может разблокироваться если ты сделаешь что-то существенное

Чтобы вытащить из `dumped` — придётся доказывать долго. Это работает не «5 хороших
сообщений → ок», а реально длительное изменение тона.

## Реалистичность

<Callout>
  Конфликт-система не «случайная». Каждый трик имеет причину которую можно прочитать. Если
  она обиделась без видимой причины — открой `conflict/log.md` и `log/YYYY-MM-DD.md`, там
  будет видно что именно зацепило.
</Callout>

## Как отключить (для тестирования)

Если тестируешь и не хочешь чтобы она обижалась:

```bash
GIRL_AGENT_DISABLE_CONFLICT=1 girl-agent
```

Или в `config.json`:
```json
{ "disableConflict": true }
```

<Callout type="warn">
  Это убивает большую часть «живости» персонажа. Не используй на постоянной основе — это
  только для тестов поведения.
</Callout>
