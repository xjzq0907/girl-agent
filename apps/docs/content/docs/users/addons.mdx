**Аддоны** — это расширения, которые меняют поведение или внешность персонажа. Формат
файла: `.gaa` (girl-agent addon) — zip-архив с `manifest.json` внутри.

Аддон может:

- добавить файлы в папку профиля (новый persona-pack, новые стикеры, новые memory-файлы)
- пропатчить `config.json` (например, поставить определённую модель LLM)
- добавить системный промпт
- инжектировать поведенческие правила

## Где искать аддоны

В **WebUI** → **Addons** → **Marketplace** — встроенный каталог. Можно установить в один
клик. Также можно положить `.gaa` файл руками в `~/.local/share/girl-agent/addons/`.

## Установка

### Через WebUI
1. Открой профиль → **Addons** → **Marketplace**
2. Найди нужный аддон → **Install**
3. Если у аддона есть **settings** — заполни их (могут быть API-ключи, токены)
4. **Apply**

### Из файла
1. Скачай `.gaa` файл
2. WebUI → **Addons** → **Install from file** → выбери файл
3. Аддон применится к **текущему профилю**

### Через CLI

```bash
girl-agent addon install ./my-addon.gaa --profile=anya
```

## Удаление

WebUI → **Addons** → ✕ напротив аддона. Удалится список установленных, но **файлы**, что
аддон скопировал в профиль, останутся. Это специально, чтобы не терять данные если ты
успел его подправить.

## Создание аддона

См. [Developer → Addon format](/docs/developers/addon-format) и [Addon development](/docs/developers/addon-development).

Краткая структура:

```
my-addon/
├── manifest.json
├── README.md
├── config.patch.json     # опционально
└── files/
    ├── persona.md
    ├── speech.md
    └── stickers/
```

И собрать:

```bash
girl-agent addon pack ./my-addon
# создаст my-addon.gaa
```

## Безопасность

<Callout type="warn">
  Аддоны **могут содержать любой Markdown / JSON**. Они не выполняют код, но могут:
  - переопределить `persona.md` (другой характер)
  - вписать в `boundaries.md` всё что угодно
  - подменить LLM-провайдер

  **Ставь аддоны только из доверенных источников.** В WebUI рядом с каждым аддоном
  указан автор и счётчик загрузок. У community-аддонов нет верификации содержимого.
</Callout>

## Что аддоны **не могут**

- Выполнять произвольный код (JS/Python)
- Получать доступ к твоему Telegram-токену кроме как через `manifest.settings`
- Лазить в чужие профили
- Отправлять данные наружу без твоего согласия

## Известные категории

- **Persona-packs** — готовые характеры (тип «офисная», «студентка», «творческая»)
- **Sticker-packs** — наборы стикеров для разных настроений
- **Behavior-patches** — изменения в поведении (более холодная / более тёплая)
- **Theme-packs** — наборы внешности WebUI
- **Integration** — связки с другими сервисами (например, погода в её сообщениях)
- **LLM presets** — кастомные провайдеры LLM с предзаполненным baseURL
- **Voice/TTS** — добавление голосовых от неё через ElevenLabs/OpenAI TTS
