`girl-agent` моделирует **жизнь** персонажа: она просыпается, бывает онлайн, занята,
ложится спать. Это не один бесконечный цикл «ответил → ждёт». У неё есть **присутствие**.

## Состояния присутствия

| Состояние    | Что делает                                                          |
| ------------ | ------------------------------------------------------------------- |
| `online`     | Активна, отвечает по таймингу профиля общения                        |
| `dnd`        | Занята (по `busySchedule`) — отложит ответ до конца окна             |
| `sleep`      | Спит — пишет утром, или ночью с шансом `nightWakeChance`             |
| `offline`    | Не запущен `girl-agent` процесс                                      |

Текущее присутствие видно в WebUI на карточке профиля и через `:status` в чате.

## Сон

### Часы

Задаются в конфиге:

```json
{
  "sleepFrom": 23,
  "sleepTo": 8
}
```

- `sleepFrom: 23` — ложится в 23:00
- `sleepTo: 8` — встаёт в 8:00
- Окно может пересекать полночь (как тут)

В её часовом поясе. Если профиль на `Europe/Moscow` — это 23:00 по Москве, а ты пишешь
из США — она «уже спит», даже если у тебя обед.

### Что происходит во сне

- Входящие сообщения **получает**, но **не отвечает сразу**
- С шансом `nightWakeChance` (по умолчанию 5%) может проснуться и ответить
- Утром после `sleepTo` она читает накопленные сообщения и отвечает — иногда подряд, иногда
  по одному с задержками
- Может извиниться: «бл я спала, не видела»

### Принудительная побудка

Если **очень надо** разбудить — отправь в чат команду:

```
:wake
```

Это команда оператору-движку, не текст для девушки. Она просыпается **на эту сессию**.
Поведение:

- В bot-режиме: ответит как обычно с задержкой профиля
- В userbot-режиме: уйдёт «онлайн», ответит, и через какое-то время снова «уснёт»

<Callout type="warn">
  `:wake` — это **ты как пользователь хочешь её разбудить**. Если она потом будет
  раздражена («ты меня разбудил») — это норма поведения, не баг.
</Callout>

### Аналоги `:wake`

- WebUI → профиль → **кнопка Wake**
- CLI:
  ```bash
  girl-agent --slug=anya --wake
  ```
- API: `POST /api/profiles/:slug/wake`

## Дневное расписание (`busySchedule`)

Окна когда она занята: пары, работа, тренировка, готовка. В эти окна — режим **dnd**:
не пишет сама, может прочитать твоё сообщение, но ответит позже.

Пример:

```json
{
  "busySchedule": [
    { "label": "пары",        "days": ["mon", "tue", "wed", "thu", "fri"], "from": "09:00", "to": "16:00", "checkAfterMin": [10, 40] },
    { "label": "тренировка",  "days": ["mon", "wed", "fri"],               "from": "18:30", "to": "20:00" },
    { "label": "сериал",      "days": ["sun"],                              "from": "20:00", "to": "22:00" }
  ]
}
```

Поля:

- `label` — что именно (используется в её отмазках: «у меня пары», «я на тренировке»)
- `days` — дни недели (`mon`/`tue`/.../`sun`). Если пусто — каждый день
- `from`, `to` — время в её tz, `HH:MM`
- `checkAfterMin` — диапазон минут, через которые она может всё-таки заглянуть в тг
  и ответить коротко (например, на паре под партой)

Окна редактируются в **WebUI → Config → Busy schedule** или прямо в `config.json`.

## Heartbeat и проактивное поведение

Каждые ~30 секунд срабатывает **online-tick** — внутренний цикл, в котором она может:

- Перечитать `agenda/` и решить, что пора что-то написать
- Проверить «открытые темы» (open-loops) — если ты что-то сказал и она не ответила, может
  всплыть тема
- Поменять presence (выйти онлайн / уйти оффлайн как живой человек)

Это всё **не равномерно**. Если она «спит» — тики идут, но она просто игнорит. Если
«занята» — может ответить очень кратко в окне `checkAfterMin`.

## Как это выглядит снаружи

В **userbot-режиме** у тебя в Telegram реально:

- Видишь её последнее посещение «час назад», «недавно», «онлайн»
- «Печатает» — настоящий индикатор, появляется когда она готовит ответ
- Прочитано — настоящее, когда движок прочитает её сообщение через API

В **bot-режиме** этого не видно (Telegram Bot API не даёт), но всё равно она отвечает с
такими же задержками — просто без визуальных индикаторов.

## Настройка в визарде

В мастере (или потом в **Config**):

- **Sleep from / Sleep to** — ползунки 0-23
- **Night wake chance** — слайдер 0..1
- **Busy schedule** — добавляешь слоты как в календаре

Если хочешь, чтобы она была **«всегда онлайн»** — поставь `sleepFrom = sleepTo` и пустой
`busySchedule`. Но это **сразу убивает реалистичность** — не рекомендуется.
