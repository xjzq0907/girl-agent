В Telegram есть два способа автоматизации: **Bot API** (через @BotFather) и **MTProto
Userbot** (твой полноценный аккаунт через api_id/api_hash). `girl-agent` поддерживает оба.

## Bot-режим

### Что это
Стандартный Telegram-бот. Пользователь видит его как **бота**: префикс «через @YourBot»,
синяя плашка «бот», нет «онлайн», ограниченные реакции.

### Плюсы
- ✅ **Безопасность:** Telegram не банит, всё в рамках API
- ✅ **Простая регистрация:** один запрос к @BotFather, токен на месте
- ✅ **Лёгкий деплой:** работает на любом сервере, в Docker, на VPS, в Termux
- ✅ Не нужен телефон — токена достаточно
- ✅ Можно сразу несколько ботов

### Минусы
- ❌ **Виден статус «бот»** — реалистичность ограничена
- ❌ Не показывает «онлайн» / «печатает» как живой
- ❌ Нет «прочитано» — Telegram Bot API его не транслирует
- ❌ Не может первым писать пользователю, пока он сам не начнёт диалог
- ❌ Не может ставить кастомные реакции, удалять старые сообщения как живой

### Когда выбирать
- Тестируешь / просто пробуешь
- Деплоишь публично, не нужна максимальная реалистичность
- Не хочешь возиться с api_id/sessions

### Как получить токен

1. Telegram → [@BotFather](https://t.me/BotFather)
2. `/newbot`
3. Имя бота (например, «Аня»)
4. Username (заканчивается на `_bot`, например `anya_irl_bot`)
5. BotFather пришлёт токен вида `1234567890:ABCdef...`
6. *(опционально)* `/setname`, `/setdescription`, `/setuserpic` чтобы кастомизировать

### Опции в @BotFather
- `/setprivacy` → **Disable** — иначе бот в группах видит только реплаи и команды
- `/setjoingroups` → **Disable** — если не хочешь, чтобы её добавляли в группы
- `/setcommands` — можно не задавать, у нас другая логика команд

## Userbot-режим (MTProto)

### Что это
Это **твой обычный Telegram-аккаунт**, которым управляет `girl-agent`. Снаружи неотличимо
от живого человека: «онлайн», «печатает», прочитано, реакции, удаление сообщений, всё.

### Плюсы
- ✅ **Максимальная реалистичность** — выглядит как обычный аккаунт
- ✅ Показывает «онлайн», «печатает», прочитано
- ✅ Может первой писать любому в контактах
- ✅ Может **ставить реакции** (включая кастомные)
- ✅ Может **редактировать и удалять** сообщения как человек
- ✅ Видит уведомления, может ответить на любой чат

### Минусы
- ⚠️ **Риск бана.** Telegram может посчитать активность подозрительной. Особенно если
  будешь массово рассылать незнакомым.
- ⚠️ Нужен **отдельный телефонный номер** — личный не используй
- ⚠️ В РФ MTProto иногда блокируется → нужен прокси или WSS-режим (включён по умолчанию)
- ⚠️ Если аккаунт забанят — данные не теряются, но **придётся регистрировать новый**

### Когда выбирать
- Нужна максимальная реалистичность (тестируешь как ИИ-агента)
- У тебя есть отдельный тестовый номер
- Готов следить за активностью и не злоупотреблять

<Callout type="error">
  **Используй ОТДЕЛЬНЫЙ тестовый аккаунт.** Никогда не подключай свой основной Telegram-
  аккаунт. Если Telegram посчитает поведение подозрительным — забанит, и ты потеряешь
  всю переписку, чаты и контакты.
</Callout>

### Как получить api_id и api_hash

1. Перейди на [my.telegram.org](https://my.telegram.org) и залогинься (понадобится код)
2. Зайди в **API development tools**
3. Создай приложение:
   - App title: любое (например, «girl-agent»)
   - Short name: латиница (например, `girlagent`)
   - Platform: **Desktop**
4. Получишь:
   - `api_id` — целое число (например, `1234567`)
   - `api_hash` — длинная hex-строка
5. Сохрани их — потом введёшь в визарде

### Авторизация при первом запуске

После ввода `api_id`, `api_hash`, телефона:
1. Telegram пришлёт **код в активную сессию** (или SMS если её нет)
2. Введи код
3. Если у тебя 2FA — введи **облачный пароль**
4. `sessionString` сохранится в `config.json` — следующие запуски будут автоматические

### Прокси и обход блокировок

По умолчанию подключение через **WebSocket на 443** (поле `useWSS: true`) — это
встроенный обход блокировки MTProto в РФ. Если не помогает:

```bash
GIRL_AGENT_TG_PROXY=socks5://user:pass@proxy.example.com:1080 girl-agent
```

Или в WebUI на вкладке **Telegram** → **SOCKS proxy**.

### Безопасность аккаунта

В вкладке профиля **Telegram** есть:
- **Перелогиниться** — стирает sessionString, заставляет заново ввести код
- **Sign out from all** — выкидывает все сессии (если думаешь, что аккаунт украли)
- **Test connection** — проверяет, что Telegram отвечает

## Сравнительная таблица

| Параметр                       | Bot                  | Userbot                       |
| ------------------------------ | -------------------- | ----------------------------- |
| Имитация человека              | низкая               | максимальная                  |
| «Онлайн» / «печатает»          | нет                  | да                            |
| Прочитано (галочки)            | нет                  | да                            |
| Реакции                        | базовые              | полные + кастомные            |
| Может писать первым            | нет                  | да                            |
| Удаление/редактирование        | ограниченно          | как живой                     |
| Видит других в групповых чатах | только через replies | всех                          |
| Можно несколько в одном проекте | да                  | один на номер                 |
| Запуск без телефона            | да                   | нет                           |
| Риск бана                      | нет                  | средний                       |
| Сложность настройки            | низкая               | средняя                       |

---

## Переключение между режимами

Можно ли в одном профиле переключать bot↔userbot? **Нет.** Память персонажа сохранится, но
история сообщений в Telegram-аккаунтах разная (бот ≠ userbot). Лучше создать **второй
профиль** в другом режиме, скопировав в него `persona.md`, `speech.md`, `boundaries.md`.
