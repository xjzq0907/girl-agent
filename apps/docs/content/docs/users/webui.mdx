После запуска `girl-agent` поднимает локальный веб-интерфейс — **WebUI** — на
`http://localhost:3000`. Это **главный способ управления**: тут визард, профили, логи,
маркетплейс аддонов, настройки. Терминал нужен только чтобы стартануть процесс.

## Как открыть

| Случай                                | Адрес                                                  |
| ------------------------------------- | ------------------------------------------------------ |
| локально на компьютере                | `http://localhost:3000`                                |
| с телефона по локалке                 | `http://192.168.x.x:3000` (запусти с `--host=0.0.0.0`) |
| Docker (с проброшенным портом)        | `http://localhost:3000`                                |
| удалённый сервер                      | через reverse proxy на `GIRL_AGENT_PUBLIC_URL`         |
| Termux                                | браузер на том же телефоне → `http://localhost:3000`   |

### Свой порт и хост

```bash
girl-agent --port=8080
girl-agent --host=0.0.0.0           # слушать на всех интерфейсах
girl-agent --no-browser             # не открывать браузер автоматически
```

### За reverse-proxy

Если за nginx/Caddy/Cloudflare Tunnel — задай **публичный URL** через env, иначе
OAuth-редиректы для LLM сломаются:

```bash
GIRL_AGENT_PUBLIC_URL=https://my-girl.example.com girl-agent --host=0.0.0.0
```

<Callout type="warn">
  По умолчанию WebUI **не имеет авторизации**. Не выставляй его в публичный интернет без
  HTTPS + базовой защиты (basic-auth на nginx, allowlist по IP). В WebUI можно прочитать
  токены и api-ключи.
</Callout>

## Что есть в WebUI

### 1. Список профилей (главный экран)

Можно держать **несколько персонажей** параллельно. Каждый — отдельный профиль с
независимой памятью, конфигом и токеном.

На каждой карточке:
- имя, возраст, национальность, стадия
- кнопка **«Запустить»** / **«Остановить»**
- иконка статуса (онлайн/спит/занята/оффлайн)
- быстрые действия: настройки, лог, удалить

### 2. Мастер первой настройки

При первом запуске или при добавлении нового профиля автоматически открывается визард:
выбор режима TG, провайдера LLM, ключей, имени и стадии. Подробнее: [Setup Wizard](/docs/users/setup-wizard).

### 3. Профиль — вкладки

После открытия профиля доступны вкладки:

#### Overview
Общий статус: какая стадия, какие счётчики, во сколько спит, что сейчас делает (если на занятиях / на работе по расписанию).

#### Config
Можно поменять любые настройки **без рестарта** (некоторые требуют перезапуска — UI
подскажет): имя, возраст, часовой пояс, профиль общения, стадию, расписание сна,
склонность к игнору.

#### Memory
Просмотр и редактирование md-файлов памяти:
- `persona.md` — кто она по характеру, что любит
- `speech.md` — как говорит, словечки, манера
- `boundaries.md` — что её бесит и обижает
- `communication.md` — стиль общения (генерируется от профиля)
- `long-term.md` — долгая память на годы
- `memory/long-term.md`, `memory/facts.md`, `memory/uncertain.md` — детали памяти
- `relationship/timeline.md` — хронология отношений
- `time/open-loops.md`, `time/promises.md` — открытые темы и обещания

[Что значит каждый файл →](/docs/users/memory)

#### Logs
Реалтайм-лог переписки: входящие, исходящие, причины игнора, изменения счётчиков. Можно
смотреть прошлые дни.

#### Agenda
Список запланированных проактивных сообщений: что и когда она планирует написать сама.
[Подробнее →](/docs/users/agenda).

#### LLM
Сменить модель/провайдер на лету, протестировать ключ (кнопка **Test**).

#### Telegram
Поменять токен бота, перелогиниться userbot'ом, подключить SOCKS-прокси для MTProto.

#### Addons
Установленные аддоны и маркетплейс. [Подробнее →](/docs/users/addons).

#### Persona
Регенерация персонажа: можно перегенерировать имя, внешность, характер, манеру речи.
Использует LLM, занимает 20-60 секунд.

#### Generate
Утилиты: сгенерить пример переписки на старте, заполнить open-loops по образцу.

### 4. Глобальные настройки

В правом верхнем углу — шестерёнка:
- темы (светлая/тёмная/auto)
- язык интерфейса
- сменить data-dir
- глобальные пресеты LLM (если ставишь свой baseURL)

## Безопасный публичный доступ через reverse-proxy

Пример nginx (Caddy/Traefik работают аналогично):

```nginx
server {
  listen 443 ssl http2;
  server_name my-girl.example.com;
  ssl_certificate /etc/letsencrypt/live/my-girl.example.com/fullchain.pem;
  ssl_certificate_key /etc/letsencrypt/live/my-girl.example.com/privkey.pem;

  auth_basic "girl-agent";
  auth_basic_user_file /etc/nginx/.htpasswd;

  location / {
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header X-Real-IP $remote_addr;
    proxy_read_timeout 86400s;
  }
}
```

Запусти `girl-agent` так:
```bash
GIRL_AGENT_PUBLIC_URL=https://my-girl.example.com girl-agent --host=127.0.0.1
```
