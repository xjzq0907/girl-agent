`girl-agent` хранит память в **обычных Markdown-файлах** в папке профиля. Не в БД — это
осознанное решение: ты можешь открыть, прочитать, поправить руками. Это сильно повышает
прозрачность и даёт полный контроль.

## Где лежит

По умолчанию (Linux):
```
~/.local/share/girl-agent/data/<slug>/
```

Можно переопределить через `GIRL_AGENT_DATA=/путь/куда/угодно`. См. [Backup & Restore](/docs/users/backup-restore).

## Структура папки профиля

```
data/anya/
├── config.json            # настройки: LLM, TG, стадия, расписание
├── persona.md             # кто она по характеру
├── speech.md              # как говорит
├── boundaries.md          # что бесит, что нельзя
├── communication.md       # стиль общения (генерится из профиля)
├── long-term.md           # обзор отношений на верхнем уровне
├── relationship.md        # текущая стадия + счётчики
├── memory/
│   ├── long-term.md       # ключевые факты и события
│   ├── facts.md           # маленькие факты (любит чай, не любит лук)
│   ├── uncertain.md       # то в чём не уверена
│   ├── shared-cross-chat.md # упоминания общих знакомых, гипертекст
│   ├── daily/
│   │   └── 2026-05-15.md  # дневной summary
│   └── episodes/
│       └── 2026-05-15-park.md # эпизод (свидание, ссора, важный разговор)
├── log/
│   ├── 2026-05-15.md      # сырой лог переписки за день
│   └── ...
├── time/
│   ├── open-loops.md      # темы которые не закрыли
│   ├── promises.md        # её и твои обещания
│   └── agenda.md          # запланированные проактивные сообщения
└── conflict/
    ├── unresolved.md      # незакрытые конфликты
    └── log.md             # хронология конфликтов
```

## Главные файлы для редактирования

### `persona.md`
Кто она по характеру. Хобби, привычки, как себя ведёт, какие интересы.

Пример:
```markdown
Аня, 22, Москва. Учится на психолога в МГППУ.

Характер: спокойная, ироничная, временами язвительная. Не любит когда давят.

Хобби: бег, читает Достоевского и Бротигана, ходит на DnB-вечеринки иногда.

Привычки: пьёт чай с лимоном, не ест мясо, спит в широких футболках.
```

Можно писать **в любом стиле**, как угодно длинно. Это идёт в системный промпт.

### `speech.md`
Как она разговаривает. Сленг, словечки, манера.

Пример:
```markdown
Часто:
- начинает с "не" в качестве усиления: "не блин, серьёзно?"
- ставит "(" / ")" вместо смайлов
- "хз" вместо "не знаю"
- "лю" в конце сообщений когда тепло
- иногда матерится мягко: "бл", "пздц", "ох"

Никогда:
- не пишет с заглавной
- не ставит много восклицательных
- не использует "rofl", "lol"
- не пишет "ахаха" длинно (только "хах" или "хах)" )
```

### `boundaries.md`
Что её бесит, что нельзя.

Пример:
```markdown
- Не любит когда о ней говорят как "девочка моя"
- Не терпит указаний что делать без её просьбы
- Раздражается на пассивную агрессию
- Никогда не делится номером телефона
- Если что-то напоминает бывшего — закрывается
```

### `communication.md`
Стиль общения. Генерится автоматически из выбранного [профиля общения](/docs/users/communication-profiles)
(Нормальная / Милая / Альтушка / Залипала / Болтушка), но можно подправить руками. Это
**самый влиятельный** файл на тон ответов.

## Память, которую она ведёт сама

Она сама дописывает в эти файлы по ходу диалога — это часть behavior-tick.

### `memory/long-term.md`
Главное что было: ваши важные разговоры, события, повороты.

### `memory/facts.md`
Маленькие факты: «он работает программистом», «у него кот Маркиз», «он живёт в Москве».
Используется чтобы не забывать детали.

### `memory/uncertain.md`
То в чём не уверена. «Кажется он сказал что играет на гитаре, не помню точно».

### `memory/daily/<date>.md`
**Дневной summary** — пересказ дня, который генерится автоматически в момент перехода
session-day. Объединяет важные эпизоды + новые факты + изменения настроения.

### `memory/episodes/<slug>.md`
Длинные **эпизоды** — отдельные важные разговоры в полной форме. Сюда попадают:
- свидания
- ссоры
- ключевые признания
- разговоры о будущем

### `time/open-loops.md`
**Открытые темы**: ты обещал прислать фото — она помнит и может напомнить. Ты сказал
«потом расскажу» — это open-loop.

### `time/promises.md`
Обещания (с твоей и её стороны), с deadline где есть.

## Сырая переписка (`log/`)

В `log/YYYY-MM-DD.md` лежит **сырой** лог переписки за день. Это **источник правды** —
сюда дописывается каждое входящее и исходящее сообщение, плюс системные пометки (причины
игнора, изменения счётчиков, ошибки).

Формат строки:
```
[2026-05-15T19:32:01.234Z] он(123456): привет, как ты?
  -> она: норм, ты как ) <!--from:123456-->
[2026-05-15T19:35:14.871Z] он(123456): а что делаешь
   ignore reason: too soon after reply, replyDelay drift
```

Лог никогда не редактируй вручную — это исторические данные.

## Просмотр и редактирование

### WebUI

Открой профиль → вкладка **Memory**. Слева — список файлов, справа — markdown-редактор с
preview. Изменения подхватываются движком **в течение 30-60 секунд** (не требуется рестарт).

### Файловый менеджер

Просто открой `~/.local/share/girl-agent/data/anya/persona.md` в VSCode/Obsidian/чём угодно
и редактируй. После сохранения — то же самое: подхватится в следующий тик.

### Команды в чате

```
:persona    # показать текущий persona.md
:log        # последние 3000 символов лога за сегодня
:log 2026-05-10  # лог конкретного дня
```

## Размер контекста

В системный промпт **не загружается всё подряд** — это убьёт стоимость. Загружается:

- весь `persona.md` + `speech.md` + `boundaries.md` + `communication.md`
- последние **30 ходов** диалога
- **summary** последнего дня + текущие open-loops + facts
- релевантные daily-summaries (по поиску ключевых слов из текущего сообщения)
- релевантные эпизоды (если матчат тему)

Размер контекста — ~6-10К токенов в обычной ситуации, ~20К на крупных моделях.

## Бэкап и приватность

<Callout type="warn">
  **Папка `data/` содержит:** твоё имя, твой номер (в userbot-режиме), всю переписку,
  токены TG, ключ от LLM. Это **по сути дамп аккаунта**. Защити:
  - не коммить в git
  - на VPS — права `700`
  - бэкап шифруй (например, `tar+age` или `rclone+encrypt`)
</Callout>

См. [Backup & Restore](/docs/users/backup-restore) и [Security & Privacy](/docs/users/security-privacy).
