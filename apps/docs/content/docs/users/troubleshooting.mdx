## WebUI не открывается

### Не приходит на `http://localhost:3000`
- Проверь, что процесс **реально запущен**: в терминале должно быть `WebUI запущен`
- Если в логе ошибка `EADDRINUSE` — порт занят. Запусти на другом:
  ```bash
  girl-agent --port=3030
  ```
- Если ты в Docker — пробросил ли порт? Должно быть `-p 3000:3000`

### С телефона по локалке не открывается
- Запусти с `--host=0.0.0.0`:
  ```bash
  girl-agent --host=0.0.0.0
  ```
- Узнай IP машины и открой `http://192.168.x.x:3000` на телефоне (он должен быть в той же
  Wi-Fi сети)
- Если есть firewall — разреши порт

### За nginx ломаются редиректы
- Задай `GIRL_AGENT_PUBLIC_URL=https://your-domain.com`
- Для WebSocket добавь `proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade";`

## LLM не отвечает

### `401 Unauthorized`
- Ключ невалидный
- Не тот провайдер (например, OpenAI-ключ суёшь в Anthropic)
- Зайди на сайт провайдера → проверь, что ключ ещё активен
- Жмёшь **Test** в WebUI рядом с полем ключа — он сделает один реальный запрос

### `429 Too Many Requests`
- Превысил rate limit. Подожди или смени тариф
- Если это часто — смени модель на менее загруженную
- Для бесплатных тиеров (Gemini, Groq) — часто бывает

### `Insufficient quota`
- Баланс кончился, пополни

### `Connection refused` / `ECONNREFUSED`
- Если это **Ollama / LM Studio** — проверь, что сервис запущен:
  ```bash
  curl http://localhost:11434/api/tags   # для Ollama
  curl http://localhost:1234/v1/models   # для LM Studio
  ```
- Если **прокси/VPN** — проверь, что нужный сервис проходит

### `Model not found`
- Опечатка в имени модели
- Модель отключена у провайдера
- В дропдауне WebUI выбери из списка, не вводи руками

### Очень медленные ответы (>30 сек)
- Прямой OpenAI/Anthropic из РФ может быть медленным без VPN
- Попробуй ClaudeHub (через РФ)
- Или Groq — быстрее всего
- Или локально через Ollama (моментально, но качество ниже)

## Telegram не подключается

### Bot-режим: `Unauthorized` от Bot API
- Токен невалидный или из неправильного бота
- Возьми токен у @BotFather заново
- Не должно быть пробелов и переносов в токене

### Userbot: `PHONE_NUMBER_INVALID`
- Номер не в международном формате — должен быть `+79991234567`

### Userbot: `PHONE_CODE_INVALID`
- Код от Telegram неверный
- Бывает что Telegram отправил несколько SMS — введи **последний**

### Userbot: `SESSION_PASSWORD_NEEDED`
- На аккаунте включена 2FA — введи облачный пароль

### Userbot: `AUTH_KEY_UNREGISTERED`
- sessionString старый / на нём вышли из всех сессий
- Перелогинься: WebUI → Telegram → **Sign in again**

### MTProto блокируется (РФ)
- По умолчанию используется WSS-обход (порт 443). Если **не помогает**:
  ```bash
  GIRL_AGENT_TG_PROXY=socks5://user:pass@proxy:1080 girl-agent
  ```
- Возьми любой работающий SOCKS5 (свой VPN-сервер тоже даст SOCKS-прокси)

### Бот не отвечает в группе
- В @BotFather: `/setprivacy` → **Disable**
- Иначе бот видит только реплаи на свои сообщения и команды

### Userbot отвечает не на все чаты
- Проверь `privacy: "owner-only"` в config — если стоит, отвечает только `ownerId`
- Поменяй на `allow-strangers` если хочешь чтобы отвечала всем

## Поведение странное

### Игнорит всё подряд
- Проверь стадию: на `tg-given-cold` шанс игнора 65% (это нормально)
- Проверь `:debug score` — если annoyance > 50, есть конфликт
- Используй `:why` — оно покажет причину последнего решения

### Не помнит что я говорил
- Проверь `memory/facts.md` — там должны быть факты
- Если факта нет — возможно она его не считала важным
- Можно дописать в `facts.md` руками: «он живёт в Москве» / «работает программистом»

### Пишет слишком одинаково
- [Профиль общения](/docs/users/communication-profiles) слишком сухой — поменяй на **Милая** или **Болтушка**
- Или подправь `speech.md` — добавь разнообразия в манеру речи
- Или смени модель на более качественную (Sonnet / GPT-5 вместо Haiku / mini)

### Слишком эмоциональная / истеричная
- Стадия слишком высокая для текущей истории — понизь
- В `boundaries.md` пропиши что злит, она будет меньше реагировать
- Можно поменять [профиль общения](/docs/users/communication-profiles) с **Милая** / **Залипала** на **Нормальная** / **Альтушка**

### Внезапно «бросила» / `dumped`
- Случилось пересечение порогов annoyance/cringe и stage-transition сработал автоматически
- Откати: `:stage dating-stable` (или какая была)
- Если хочешь чтобы такого больше не было — `GIRL_AGENT_NO_AUTO_STAGE=1`

### Слишком часто пишет первой
- Поменяй **инициативу** на «редко» в [профиле общения](/docs/users/communication-profiles)
- В WebUI → Agenda → **Clear all** и пусть копится с нуля

### Совсем не пишет первой
- Стадия низкая (1-2)
- Или инициатива стоит на «редко»
- Или у неё всё время сон / занятость (проверь расписание)

## WebUI глючит

### Не показываются обновления в логе
- Перезагрузи страницу
- Если не помогает — WebSocket заблокирован (за реверс-прокси?)
- Проверь консоль браузера на ошибки

### Сохранение конфига откатывается
- Кто-то ещё открыл WebUI и поменял настройки одновременно
- Перезагрузи страницу и сохрани заново

### «Failed to fetch»
- Процесс упал. Проверь терминал
- Перезапусти: `girl-agent` (или `systemctl restart girl-agent`)

## Установка

### `Node.js version too old`
- Нужен Node.js 18.18 или новее
- Обнови: [nodejs.org](https://nodejs.org/) или через nvm/volta

### `Permission denied` на Linux
- Не от sudo. Не от sudo. **Не от sudo.**
- Если глобальная установка: `npm install -g girl-agent` (от своего юзера) или используй
  prefix:
  ```bash
  npm config set prefix ~/.npm-global
  export PATH=~/.npm-global/bin:$PATH
  npm install -g girl-agent
  ```

### Termux: `pkg install nodejs` ставит старую версию
```bash
pkg install nodejs-lts
```

### Docker: `ENOSPC` при сборке
- Не хватает диска. Очисти Docker:
  ```bash
  docker system prune -a
  ```

## Диагностика

Полезные команды для отладки:

```bash
# текущая версия и платформа
girl-agent --version

# список профилей и где они лежат
girl-agent --list

# проверить миграции данных
girl-agent update --verbose

# реалтайм-лог одного профиля
tail -f ~/.local/share/girl-agent/data/anya/log/$(date +%F).md

# проверить health-эндпоинт WebUI
curl http://localhost:3000/api/system/health

# проверить версию Node
node --version
```

В чате с ботом:

```
:status      # общий статус
:debug score # детально по счётчикам
:debug presence # презенс + следующий тик
:why         # почему она так ответила
:log         # сегодняшний лог
```

## Сообщить о баге

Если ничего не помогло:

1. Собери:
   - вывод `girl-agent --version`
   - `node --version`
   - ОС и версия
   - последние 100-200 строк лога (но **без api-ключей и токенов** — поищи и затри)
   - что делал и что ожидал

2. Открой issue: [github.com/TheSashaDev/girl-agent/issues](https://github.com/TheSashaDev/girl-agent/issues)

3. Опиши шаги воспроизведения

</Callout>
