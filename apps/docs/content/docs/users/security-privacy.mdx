`girl-agent` — **локальное приложение**. Все данные у тебя на машине. Но есть три места,
где данные могут уйти:

1. **LLM-провайдер** — отправляются твои сообщения + системный промпт
2. **Telegram-серверы** — отправляются все сообщения (как и в обычном Telegram)
3. **WebUI** если выставлен в публичный интернет

Разберёмся по каждому.

## Что НЕ собирается

- Никакой телеметрии (ни Sentry, ни PostHog, ни Google Analytics)
- Никаких метрик использования
- Никаких автоматических отчётов
- Нет «облака girl-agent»
- Нет email-регистрации

## Что отправляется к LLM-провайдеру

При **каждом твоём сообщении**:
1. Твой текст
2. История диалога (последние ~30 ходов)
3. `persona.md`, `speech.md`, `boundaries.md`, `communication.md`
4. Сводки и факты (memory/facts.md, daily-summaries)
5. Текущее presence/dailyLife state
6. Метаданные (твоё имя если знает, твой owner-id)

LLM-провайдер **видит всё это**. Они могут логировать запросы. У них своя retention
policy:

| Провайдер     | Логи запросов                                           |
| ------------- | ------------------------------------------------------- |
| OpenAI        | 30 дней (для abuse-monitoring), не используется для обучения если на платном тарифе |
| Anthropic     | 30 дней, не используется для обучения                   |
| ClaudeHub     | по их policy, см. сайт                                  |
| Groq          | временные логи, не используются для обучения            |
| DeepSeek      | в соответствии с китайским законодательством            |
| Gemini        | используется для обучения если **не на платном** API    |
| Ollama        | **локально**, никуда не уходит                          |
| LM Studio     | **локально**, никуда не уходит                          |

Если **критично** не показывать переписку никому — используй **локальный LLM** (Ollama
или LM Studio).

## Что отправляется к Telegram

То же, что и в обычном Telegram-чате — все сообщения проходят через серверы Telegram.
Telegram не шифрует обычные чаты end-to-end (секретные — да, но они работают только в
официальном клиенте).

Иными словами: если ты не хочешь чтобы Telegram-сервер хранил переписку — `girl-agent`
не поможет (как и любой другой клиент).

## Локальные данные

В папке `data/<slug>/`:
- `config.json` — содержит **api-ключ LLM**, **TG botToken** или **sessionString**,
  телефон, **api_hash**
- `log/*.md` — вся переписка в открытом виде
- `memory/*` — все факты о тебе

<Callout type="error">
  **`config.json` — это твой ключ от Telegram-аккаунта и LLM-API.** Если злоумышленник
  получит файл, он получит и аккаунт TG (в userbot-режиме), и ключ к биллингу LLM. Защити:
  - права `chmod 600 config.json` (Linux/macOS)
  - не клади в git
  - бэкап — только зашифрованный
  - на multi-user VPS — внимательно с правами папки
</Callout>

## Защита WebUI

По умолчанию WebUI слушает на **`127.0.0.1`** — недоступен из вне. Если ты разворачиваешь
WebUI наружу:

### Базовая защита (минимум):
1. HTTPS с реальным сертификатом
2. Basic-auth на nginx/Caddy
3. IP allowlist на ту, что у тебя
4. `GIRL_AGENT_PUBLIC_URL=https://yours.tld` чтобы редиректы работали

### Расширенная защита:
1. Через VPN (WireGuard / Tailscale) — никакого публичного доступа вообще
2. Cloudflare Access / Tunnel для OAuth-аутентификации
3. Файервол на уровне OS

<Callout type="warn">
  В WebUI можно прочитать api-ключи LLM, TG-токены, всю переписку, и поменять конфиг.
  **Не выставляй без авторизации в публичный интернет.**
</Callout>

## Защита от перехвата ключей

API-ключи LLM попадают в:
- `config.json` (на диске)
- Запросы к API провайдера (по HTTPS, но ключ в заголовке)
- Логи WebUI (если включена debug-кнопка)
- Системные переменные (если задаёшь через env)

Рекомендации:
- Для CI/Docker — **через секреты**, не через `-e GIRL_AGENT_API_KEY=...` в plain
- На локальной машине — через `.envrc` (direnv) или менеджер паролей
- В CI — через `${SECRETS.OPENAI_KEY}`
- На VPS — `chmod 600 ~/.local/share/girl-agent/data/<slug>/config.json`

## Userbot — особый риск

Когда логинишься userbot'ом:

1. `sessionString` сохраняется в `config.json` — это **полный доступ к аккаунту**
2. Если sessionString украдут — могут зайти под твоим аккаунтом, не зная пароля
3. Telegram пометит вход с нового IP — попросит подтверждение
4. Но **уже залогиненная сессия** работает без подтверждений

**Что делать:**
- Используй отдельный тестовый аккаунт
- Включи 2FA облачным паролем
- Регулярно `Sign out from other sessions` в Telegram
- Если думаешь что украли — `Sign out all` и перелогинься

## Анти-AI-детекция

В `girl-agent` есть **anti-AI-detection** слой:
- Не пишет «как ИИ»
- Не использует характерные обороты («как языковая модель», «я понимаю ваш вопрос»)
- Сохраняет манеру речи из `speech.md`
- Опечатки, паузы, лёгкие реалистичные ошибки

Но это **не невидимка**. Если человек **очень внимательно ищет** — может заметить
закономерности (особенно если использовать в высокоставочных контекстах). Не полагайся на
это как на «полную невидимость».

## Удаление данных

Если хочешь полностью удалить персонажа:

```bash
girl-agent --delete-profile --profile=anya --yes
```

Или вручную:
```bash
rm -rf ~/.local/share/girl-agent/data/anya
```

В userbot-режиме также:
1. Telegram → Settings → Privacy → Active sessions → завершить сессию
2. Удалить переписку в самих Telegram-чатах
