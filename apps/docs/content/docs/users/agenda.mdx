**Agenda** — это список **запланированных проактивных сообщений**, которые она напишет
сама без твоего пинга. Это то, что отличает живой персонаж от чат-бота: она помнит, что
надо пожелать удачи перед твоим экзаменом, спросить как прошла встреча, поздравить с днём
рождения.

## Где смотреть

- **WebUI** → профиль → вкладка **Agenda** — таймлайн что и когда она планирует
- **Файл:** `data/<slug>/time/agenda.md` — сырая запись пунктов
- **В чате:** `:agenda` (если включён debug)

## Что попадает в agenda

Источники:

1. **Что ты сам сказал** — упомянул что у тебя завтра собес → она запомнит и спросит как
   прошёл
2. **Свои планы** — у неё в `persona.md` сказано «работает в кафе по выходным» → может
   написать «иду открывать кафе»
3. **Эпизоды и open-loops** — ты что-то обещал → она напомнит
4. **Дни рождения / праздники** — если в `memory/facts.md` есть «др 14 июня» — поставит
   пинг

Каждый пункт = `AgendaItem`:

```json
{
  "id": "ag-2026-05-16-1234",
  "about": "его собес в гугл",
  "userEventTime": "2026-05-17T10:00:00.000Z",
  "pingAt": "2026-05-17T09:30:00.000Z",
  "reason": "пожелать удачи перед",
  "kind": "user-event"
}
```

И ещё один следом:

```json
{
  "id": "ag-2026-05-16-1235",
  "about": "его собес в гугл",
  "userEventTime": "2026-05-17T10:00:00.000Z",
  "pingAt": "2026-05-17T15:00:00.000Z",
  "reason": "узнать как прошло",
  "kind": "user-event-followup"
}
```

## Как это работает

### Источник 1: ты упомянул событие
Она парсит твои сообщения через LLM и вылавливает события вида:
- «завтра собес» → ставит пинг
- «в субботу полечу в Питер на 3 дня» → пишет в субботу «долетел?»
- «после работы пойду в зал» → может прислать «как качаешься?»

Парсинг не идеальный — иногда ложит false positives. Можно почистить вручную в файле или
WebUI.

### Источник 2: свои планы
Из её `persona.md` + `busySchedule` движок может **сам** сгенерировать пункты agenda:
- «учится на психолога» → может написать «у меня сегодня лекция по травме, посплю позже»
- «работает в кафе пт-сб» → пишет в пт вечером «домой пришла, ноги отваливаются»

Эти пункты называются **proactive-life** и не привязаны к твоим событиям.

### Источник 3: открытые темы
Если ты что-то обещал и не сделал — она может напомнить:
- Ты сказал «потом скину видос» → через 1-2 дня «ну где видос-то»
- Ты сказал «вечером позвоню» → если не позвонил, может молча обидеться или спросить

## Когда отправляется

Тик `agenda` срабатывает каждые ~5 минут. Проверяет:
- наступило ли `pingAt`?
- она сейчас может писать (не спит, не занята)?
- последнее сообщение было достаточно давно (не спамит)?
- стадия позволяет (на `tg-given-cold` редко пишет первой)?

Если все условия да — формирует сообщение через LLM и отправляет.

## Управление

### Удалить пункт
- WebUI: кнопка ✕ напротив пункта
- Файл: открой `time/agenda.md` и удали JSON-блок руками

### Добавить вручную
В WebUI на вкладке Agenda — **Add item**. Полезно если хочешь подтолкнуть поведение:
«через час напиши “соскучилась”».

### Очистить всё
- `:reset agenda` в чате
- WebUI → **Clear all** (с подтверждением)

## Лимиты и анти-спам

- В сутки **не больше 5-7 проактивных сообщений** по умолчанию
- Если у тебя несколько диалогов идёт активно — agenda смещается, чтобы не дублировать
- В ночное время (sleep) — только если `nightWakeChance` сработает + пункт **критический**
- На стадиях 1-2 — почти никогда (она же ещё не уверена что хочет общаться)

## Если она пишет слишком много / мало

| Симптом                          | Что сделать                                      |
| -------------------------------- | ------------------------------------------------ |
| Пишет 10+ раз в день сама        | Уменьшить **инициативу** в [профиле общения](/docs/users/communication-profiles) (поставь «редко») |
| Совсем не пишет первой           | Поднять стадию или поставить **инициативу = «часто»**                                              |
| Пишет в неподходящие моменты     | Уточни расписание (она думает, что свободна)                                                       |
| Пишет про несуществующие события | Очисть `agenda.md` руками — там накопились галл                                                    |

## Файл `time/agenda.md`

Структура (JSON-блоки внутри markdown):

```markdown
<!-- girl-agent agenda v1 -->

\`\`\`json
{
  "id": "ag-2026-05-16-9341",
  "about": "его выпускной",
  "userEventTime": "2026-05-25T15:00:00.000Z",
  "pingAt": "2026-05-25T13:00:00.000Z",
  "reason": "поздравить",
  "kind": "user-event",
  "createdAt": "2026-05-16T12:00:11.213Z"
}
\`\`\`

\`\`\`json
{ "id": "...", ... }
\`\`\`
```

Можно редактировать руками между ``` блоками, но проще через WebUI.
