Кроме обычной переписки, можно отправлять **команды оператору-движку** прямо в чат — они
начинаются с двоеточия `:` и **не передаются** девушке как реплики.

<Callout type="info">
  В bot-режиме команды можно слать в чат с ботом. В userbot-режиме лучше слать **в
  саму-себя** или специальный отладочный чат — иначе она реально получит сообщение в
  Telegram.
</Callout>

## Базовые

### `:status`
Полный статус: какая стадия, счётчики, presence (спит/онлайн/занята), активная модель,
последние ивенты движка.

### `:relationship`
Короткая выжимка: `stage=convinced score={"interest":55,"trust":30,...}`.

### `:cringe`
Сколько раз ты её закринжанул + ссылка на `memory/long-term.md` и `log/`.

### `:why`
Объяснение последнего решения движка: «почему она проигнорила», «почему ответила одним
словом», «почему написала первой». Очень полезно для отладки.

### `:help`
Список всех доступных команд.

## Управление присутствием

### `:wake`
Принудительно разбудить, если она в `sleep`. Игнорирует sleep window на эту сессию.

```
:wake
```

### `:pause`
Поставить движок на паузу. Не отвечает, не делает agenda-ticks, не меняет presence.
Полезно если хочешь «заморозить» и поправить настройки.

### `:resume`
Снять паузу.

## Память и логи

### `:persona`
Показать содержимое `persona.md`.

### `:log [date] [limit]`
Сырой лог переписки. Без аргументов — сегодня, последние 3000 символов.

```
:log                 # сегодня
:log 2026-05-10      # конкретный день
:log 2026-05-10 8000 # 8К символов
```

### `:amnesia [target] [arg]`
Стереть определённую память (с подтверждением!):

```
:amnesia conflict          # очистить unresolved конфликты
:amnesia agenda            # очистить план
:amnesia daily 2026-05-10  # стереть daily summary за день
:amnesia uncertain         # очистить uncertain.md
:amnesia all               # ОПАСНО: всё кроме персоны
```

### `:reset`
Полный сброс отношений: счётчики обнуляются, стадия возвращается к начальной из config.
Персона и memory остаются.

## LLM и модели

### `:model`
Без аргументов — показать текущую LLM-модель.

### `:model <preset> <model> [api-key]`
Сменить модель на лету:

```
:model claudehub claude-opus-4.6
:model openai gpt-4o sk-...
:model ollama qwen2.5:14b
```

## Стадии

### `:stage [num или id]`
Сменить стадию вручную:

```
:stage 5                # по номеру
:stage first-date-done  # по id
:stage                  # показать текущую
```

## Отладка

### `:debug <subcommand>`
Развёрнутая отладка:

```
:debug score        # детально по счётчикам с причинами
:debug presence     # презенс + следующий тик
:debug agenda       # план + причины каждого пункта
:debug conflict     # все открытые конфликты
:debug timing       # как считаются задержки
:debug tokens       # сколько токенов было в последнем запросе
```

### `:snapshot`
Аггрегированный JSON-снимок состояния (используется десктопным враппером, но и людям
полезно): профиль, стадия, счётчики, presence.

## Стикеры

### `:sticker <mood>`
Принудительно отправить стикер по настроению:

```
:sticker happy
:sticker sad
:sticker laugh
:sticker thinking
:sticker love
```

## Завершение

### `:quit` / `:exit`
Корректно остановить движок (graceful shutdown). Эквивалент `Ctrl+C` в консоли.

## Headless / JSON-mode команды

В headless-режиме (десктоп-враппер) **тот же набор команд** работает через stdin как
NDJSON. См. [Developer → Headless mode](/docs/developers/headless-mode).

## Не все команды видны человеку

Команды:
- **не попадают** в `log/` как реплики юзера (только как `[cmd] :status`)
- **не вызывают** behavior-tick (она не «обижается» на :status)
- **не считаются** активностью пользователя

Так что можно дебажить **сколько угодно**, не ломая контекст разговора.
